1.3. Users manual

  1. Documentation of how to use the extension, how it works, how to apply it if it's a website plugin. A user manual.

  2. Language should be non-technical, explaining, using small examples.

  3. Examples: For the "News" plugin this would be a manual showing how to create the news-items, explaining the options etc.

  4. Target group: Users, Administrators or Developers in that priority. Depends on the extension.

Local Language String Tokens

Some local language strings use special tokens. They are explained in detail below:

Items[this is header 4 but additional it is set to bold!]

title.titleText[this is bold but bold is not possible yet]

%t – replaced with the item's title.

client.titleText

%t – replaced with the client's title.
